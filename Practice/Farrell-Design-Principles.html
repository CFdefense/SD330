My Design Principles

User-Centric Design

Do's

Conduct user research to understand user needs.

Create user personas to guide design based on real user data.

Involve users in testing to gather active feedback.

Ensure users feel in control throughout.

Dont's

Assume you know what users want without research.

Neglect user feedback after the initial testing phase.

Design solely based on personal choice instead of user needs.

Prioritize aesthetics over functionality.

Accessibility

Do's

Ensure sufficient color contrast for readability.

Include alternative text for images to aid visually impaired.

Use semantic HTML elements.

Provide keyboard navigation to make all elements accessable.

Dont's

Include alternative text for images to aid visually impaired.

Neglect accessibility testing with disabled users.

Rely solely on color to convey information.

Provide transcripts for multimedia content.

Consistency

Do's

Maintain a consistent visual style.

Standardize interactions for similar actions.

Use consistent terminology throughout.

Use a uniform color palette.

Dont's

Mix different styles for similar elements.

Change navigation patterns frequently.

Ignore established brand guidelines and design language

Use varying terms for the same action.

Feedback

Do's

Provide clear error and slip messages.

Display loading indicators and updates.

Provide feedback to users within half a second.

Reward positive actions with visual or audible cues.

Dont's

Leave users guessing after an interaction.

Neglect to acknowledge user input despite slipping.

Overwhelm users with TOO much feedback.

Use overwhelming pop-ups for minor actions.

Simplicity

Do's

Emphasize minimalism while reducing distractions.

Organize information logically facilitating easy navigation.

Highlight clear calls-to-action to guide users throughout.

Use simple and easy to understand language.

Dont's

Overload users with excessive information.

Hide important features within menus or dropdowns.

Use complex jargon and unclear icons

Clutter the interface with content.

Adaptability

Do's

Use progressive disclosure of information to prevent overload

Allow user customization from themes to preferences.

Test designs in different screen and browser environments.

Create adaptive typography at all screen sizes

Dont's

Ignore different user device contexts.

Make essential features device-dependent.

Assume high-speed internet for all users.

Assume users all speak the same language.
